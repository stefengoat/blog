Le projet Java Modération

Objectif du projet


Dans le cadre de l’évolution du site r3st0.fr, une application Java a été développée pour les modérateurs, destinée à être installée sur leurs postes de travail.

Cette application permet de contrôler les avis déposés par les utilisateurs, avec la possibilité de supprimer ou bloquer ceux ne respectant pas la charte d’utilisation du site

L’accès à l’application est sécurisé par un système d’authentification pour chaque modérateur.

Ce projet a été mené en équipe de quatre personnes, selon une approche collaborative

Nous avons travaillé à partir de User Story fonctionnels, avec des réunions régulières pour coordonner l’avancement et ajuster les choix techniques.

L’application repose sur la base de données existante, laquelle a pu être adaptée si nécessaire.

Le projet devait être accompagné de comptes-rendus de chacune des itérations. Vous pouvez les consulter.

Compétences


Gérer des sauvegardes : Le projet Java a été régulièrement sauvegardé sur un dépôt GitLab, permettant de conserver un historique des versions, de faciliter le travail collaboratif et d'assurer la sécurisation du code source en cas de perte de données ou d'erreurs.

Collecter, suivre et orienter des demandes : Les demandes ont été collectées et organisées à partir d’un cahier des charges détaillé, accompagné de tickets de suivi. Cela a permis de prioriser certaines tâches urgentes et de planifier le développement de l’application Java de manière structurée et efficace.

Évaluer les indicateurs de suivi d'un projet et analyser les écarts : Des daily scrum étaient organisés chaque début de semaine, accompagnés de la rédaction d’itérations hebdomadaires. Cela permettait de comparer les avancées réelles aux objectifs annoncés, d’évaluer les écarts et d’ajuster la planification en conséquence.
